What does API stand for?

API stands for Application Programming Interface.
But what does that mean?

An API is a set of rules that define how a program can communicate with another program.

Basically that means it's a way you can call code you didn't write.

Examples

  • System Libraries (windows.h, glibc)
  • User Libraries (ffmpeg, imagemagick)
  • Foriegn Function Interfaces (calling C code from Rust)
  • Web APIs (SOAP, REST)

A library is not the same thing as an API. The API is just the interface that is used to communicate with the library.
Interacting with most of these depends on the context they are being used in. Web APIs are special because the context they are being called from does not matter

What is a REST API?

REpresentational State Transfer

REST is a standard for how web APIs should be designed.

Runs on top of HTTP

  • Stateless
  • Scalable
  • Uniform

REST vs SOAP

  • Design Principles vs Protocol
  • Multiple Return Types vs XML Only

The SOAP Protocol has built in secuirty

The added security and more rigid structure makes SOAP more favorable to enterprise environements, but the freedom and light-weight nature of REST makes it ideal for smaller, simpler solutions.
