History

  • Sun creates Java
  • Microsoft creates Visual J++
  • Microsoft modifies J++ with features and integrations
  • Sun sues Microsoft
  • Microsoft creates CLR, .NET, and C#

Sun sued Microsoft because J++ was technically a different language, but was still being used under the Java branding.
CLR is cross-platform language runtime. C# was built on top of CLR and .NET along with Visual J# which was more similar to Java and was intended to help Java developers transition easier.

The C# Language

Similar looking to Java
Modern Language
Large async support

Other .NET supported Languages

  • F#
  • Visual Basic

Complilation

Initially compiled to CIL
Similar to Java's Bytecode
Compiled to machine code by CLR

CIL is the Common Intermediate Language. CLR is the Common Language Runtime.

.NET vs Framework vs Core vs Standard

.NET Framework is the original
.NET Core is the open source, cross-platform fork
.NET Standard is the required features of all .NET Implementations
.NET (by itself) is a rebrand of .NET Core that is also the end of .NET Framework

ASP.NET

Web Application Framework

Razor/Blazor

Webpage Desgin Framework
Uses HTML, CSS, and JavaScript along with C#
Blazor - Serverside rendering and WASM

Xamarin/MAUI

Cross-platform mobile application framework
Uses XML for layout
MAUI - Cross-platform with Linux

WPF/WinForms/UWP

Windows specific application framework
Mostly use XML for layout
UWP - Windows Store apps, run on Xbox

WPF is Windows Presentation Foundation framework. UWP is Universal Windows Platform.
UWP supports using HTML for layout.

Azure

Linq

Advanced collection processing

Dependency Injection

Technique for resolving dependencies in a loosely coupled way
Used with ASP.NET and EF
Basically black magic

Can get reference to a service without bloilerplate setup or creating a class level reference

Entity Framework

Implementation agnostic
